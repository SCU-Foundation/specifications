## Structural Rules

- all admin files MUST exist at the root of the /specifications/ directory  
- each specification MUST reside in its own subdirectory (ts, lp, lm, lw, lit)  
- each specification subdirectory MUST contain its own admin files and canonical specification document  
- no additional root‑level folders may be added without governance approval  
- all references MUST use repo‑scoped paths  
- all files MUST follow canonical formatting and naming conventions
